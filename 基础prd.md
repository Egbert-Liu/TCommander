# Client Manager 开发文档

## 1. 产品概述

### 1.1 产品定位

Client Manager 是一个面向开发者的多终端会话管理桌面应用，专为统一管理 AI Client（如 Clock Code、Codex 等）而设计。

### 1.2 解决痛点

开发者在日常工作时，常需要同时打开多个终端命令窗口（如 PowerShell、CMD、npm 等），当会话数量超过5个时，管理成本急剧上升，难以判断哪些会话需要确认、哪些报错、哪些处于空闲状态。

### 1.3 核心价值

Client Manager 将所有终端会话状态集中管理，以卡片形式展示每个会话的状态，支持自动检测、状态标记，并提供快捷操作和快照能力，实现工作环境的一键保存与恢复。

## 2. 核心功能模块

### 2.1 多会话管理

- 以卡片形式展示多个 PTY 终端会话
- 支持创建、关闭、切换多个终端会话

### 2.2 状态感知引擎

- 自动检测终端输出模式（yn 提示、箭头输入、进程状态等）
- 标记卡片状态：需确认、错误、待输入、空闲等
- 实时状态更新与显示

### 2.3 名称搜索

- 工具栏提供模糊匹配搜索框
- 实时过滤匹配的会话卡片
- 支持会话名称快速定位

### 2.4 快捷操作

- 卡片底部控制栏支持：
    - 一键 yn 确认
    - Ctrl+C 中断
    - 方向键导航
    - 自定义输入
    - Enter 发送内容
    - Shift+Enter 换行

### 2.5 全屏终端

- 点击卡片进入基于 xterm.js 的全屏终端视图
- 支持快捷键 S 触发全屏模式
- 完整的终端功能体验

### 2.6 分组管理

- 左侧边栏支持自定义分组
- 分组颜色标记与筛选
- 会话分组归类管理

### 2.7 快照能力

- 一键保存当前所有会话布局与状态
- 支持后续一键恢复工作环境
- 完整的会话状态持久化

### 2.8 预设系统

- 新建会话时可指定：
    - 工作目录
    - 终端类型（PowerShell/CMD）
    - 初始执行命令
- 整套配置保存为预设模板
- 支持后续一键复用

### 2.9 持久化存储

- 分组配置持久化
- 快照数据持久化
- 预设模板持久化
- 控制栏设置持久化
- 应用重装后可恢复配置

#### 核心实现细节

- **路径解析**：识别 CMD/PowerShell/PWSH 等终端路径
- **PTY 环境变量**：设置 UTF-8 编码（chcp/LC_ALL）
- **初始命令**：延迟 500ms 写入，避免命令丢失
- **输出缓冲区**：每个会话最多保存 200 行输出
- **ANSI 清理**：预览与状态检测前剥离 ANSI 转义序列
- **全屏终端**：先写入历史 buffer 再订阅实时流
- **卡片排序稳定性**：
    - 主键：createAt（创建时间，固定不变）
    - 次键：状态优先级
    - lastActivityAt 仅在状态变更时更新，不参与排序
- **安全 IPC**：关闭时防止向已销毁窗口发送消息
- **窗口扫描**：PowerShell 定时扫描外部终端窗口（1-2 秒延迟）
- **多行输入框**：Enter 发送，Shift+Enter 换行，自动扩展行高

## 4. 已知限制与注意事项

### 4.1 功能限制

- **窗口绑定**：仅支持 Windows 平台，当前阶段暂不开发
- **控制台读取**：仅能读取可见行内容，非完整历史记录
- **初始命令**：可能因 Shell 慢启动导致命令丢失
- **状态扫描**：存在 1-2 秒的延迟
- **快照功能**：不保存绑定的外部绘画（PID 重启后失效）

### 4.2 开发经验总结

1. **node-pty 选型**：使用主包（包含 Windows 预编译版本），避免使用 prebuild 变体
2. **无边框窗口拖动**：设置 webkit-app-region 为 drag，按钮区域设置为 no-drag
3. **xterm 历史回放**：先进入历史 buffer 再订阅实时流，避免丢失输出
4. **避免卡片抖动**：不要使用 lastActivityAt 作为排序键
5. **关闭错误处理**：处理 PTY 回调异常，防止 Windows 错误弹窗
6. **ANSI 处理**：确保彻底清理 ANSI 转义序列
7. **主题统一**：保持 Dark/Light 主题样式协调一致
8. **快速输入框**：使用 onKeyDown 拦截 Enter 组合键逻辑
