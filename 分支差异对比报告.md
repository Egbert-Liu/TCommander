## 二、变更详情

### 2.1 数据流优化 — `App.tsx`

**变更目的**: 解决终端内容显示延迟问题

**核心改动**:

1. **节流策略替换**: 将 `requestAnimationFrame` 单帧节流替换为 **双定时器策略**
   - **前缘**: 首个 chunk 立即 flush（0ms 延迟），确保用户操作即时响应
   - **后缘**: 连续输出时，每 16ms（一帧）flush 一次，避免过度渲染
   - 新增 `lastFlushTimeRef`、`trailingTimerRef`、`FLUSH_INTERVAL = 16` 三个变量
   - 移除原有的 `rafIdRef`

2. **卡片网格布局优化**: 最小宽度从响应式断点改为 `minmax(500px, 1fr)`
   ```
   // 旧
   <div className="grid grid-cols-1 md:grid-cols-1 lg:grid-cols-2 xl:grid-cols-3 gap-5">
   // 新
   <div className="grid gap-5" style={{ gridTemplateColumns: 'repeat(auto-fill, minmax(500px, 1fr))' }}>
   ```
   确保卡片不会过窄，终端内容有足够展示空间

3. **清理逻辑适配**: 所有引用 `rafIdRef` 的清理代码统一改为 `trailingTimerRef`
   - `handleExit` 中的 `cancelAnimationFrame` → `clearTimeout`
   - 组件卸载清理中的批量取消逻辑同步更新
   - 进入全屏时的立即 flush 逻辑同步更新

**风险评估**: 低。逻辑等价替换，仅改变了节流方式。

### 2.3 全屏终端快捷键切换 — `FullscreenTerminal.tsx`

**变更目的**: 支持 Ctrl+Alt+左右方向键在全屏模式下快速切换终端

**核心改动**:

在 Terminal 创建后，通过 `attachCustomKeyEventHandler` 注册快捷键拦截：

```typescript
terminal.attachCustomKeyEventHandler((e: KeyboardEvent) => {
  if (e.ctrlKey && e.altKey && (e.key === 'ArrowLeft' || e.key === 'ArrowRight')) {
    // 左: 上一个终端（循环）
    // 右: 下一个终端（循环）
    setActiveSession(targetSession.id)
    return false // 阻止事件继续传播
  }
  return true // 其他按键正常处理
})
```

**设计要点**:
- 使用 `attachCustomKeyEventHandler` 而非 `window.addEventListener`，确保在 xterm 获得焦点时也能正确拦截
- 循环切换：到末尾后按右回到第一个，到开头后按左回到最后一个
- `return false` 阻止快捷键被终端 PTY 捕获

**风险评估**: 低。仅在特定快捷键组合时触发，不影响正常终端输入。

### 2.7 ANSI 颜色渲染修复 — `ansiToHtml.ts`

**变更目的**: 修复卡片预览区颜色丢失的问题

**核心改动**:

1. **默认前景色始终输出**: `styleToCss` 函数新增 `defaultFg` 参数，未设置颜色时用主题前景色兜底
   ```typescript
   // 旧: 无颜色时不输出 color 属性
   if (s.color) parts.push(`color:${s.color}`)
   // 新: 始终输出 color，确保所有文本都有颜色
   parts.push(`color:${s.color || defaultFg || 'inherit'}`)
   ```

2. **修复全局正则 lastIndex 污染**: 使用独立的非全局正则 `SGR_TEST_RE` 做 CSI 过滤测试
   ```typescript
   // 旧: 直接复用全局 SGR_RE 做 test，lastIndex 状态导致偶发颜色丢失
   text = text.replace(CSI_RE, (m) => (SGR_RE.test(m) ? m : ''))
   // 新: 用独立的非全局正则测试
   const SGR_TEST_RE = /\x1b\[([\d;]*)m/
   text = text.replace(CSI_RE, (m) => (SGR_TEST_RE.test(m) ? m : ''))
   ```

3. **重置 lastIndex**: 在解析循环前显式重置 `SGR_RE.lastIndex = 0`

**风险评估**: 低。修复了已知的颜色丢失 bug，不引入新行为。

### 2.8 预设排序工具函数 — `presetSort.ts` (新增)

**变更目的**: 提取公共排序逻辑，避免 PresetsDialog 和 NewSessionDialog 重复实现

**核心改动**:
```typescript
export function sortPresets(presets: Preset[]): Preset[] {
  return [...presets].sort((a, b) => {
    const priorityDiff = (b.priority || 0) - (a.priority || 0)
    if (priorityDiff !== 0) return priorityDiff
    return (b.usageCount || 0) - (a.usageCount || 0)
  })
}
```

**风险评估**: 无。纯新增文件。
