🎨 TCommander 卡片预览方案对比

基于业界主流终端工具（Wave Terminal、Tabby、VS Code、Hyper）的设计模式，提供 4 种预览方案供选择

方案 A: 垂直滚动 + 自动换行
推荐
设计理念：最符合用户阅读习惯的方式，长行自动换行，垂直滚动查看更多内容。
参考产品：Wave Terminal、Hyper Terminal、VS Code 集成终端

优势

  • 符合自然阅读习惯，无需横向滚动
  • 内容利用率高，充分利用卡片宽度
  • 业界主流方案，用户接受度高
  • 适合查看长命令输出和日志

劣势

  • 长命令换行后可能影响可读性
  • 无法精确保持终端原始格式
  • 复制粘贴时可能包含多余换行

💡 适用场景

适合大多数使用场景，特别是查看构建日志、命令输出、错误信息等。用户无需横向滚动即可看到完整内容，阅读体验最佳。

方案 B: 水平滚动 + 不换行
当前方案
设计理念：完全保持终端原始格式，长行不换行，通过水平滚动查看。
参考产品：传统终端模拟器、tmux 预览、纯文本编辑器

优势

  • 完美保持终端原始格式
  • 长命令不会被截断或换行
  • 复制粘贴内容准确无误
  • 适合查看表格、对齐数据

劣势

  • 需要横向滚动，操作不便
  • 宽内容可能被截断看不到
  • 不符合常规阅读习惯
  • 小屏幕下体验较差

方案 C: 固定高度 + 底部淡出
视觉优先
设计理念：固定预览区高度，底部使用渐变淡出效果，暗示还有更多内容。
参考产品：macOS Terminal 预览、Slack 代码块、GitHub Gist

优势

  • 卡片高度统一，布局整齐美观
  • 淡出效果优雅，视觉体验好
  • 自动换行，阅读舒适
  • 适合快速浏览最新输出

劣势

  • 无法看到完整历史内容
  • 需要进入全屏才能查看完整输出
  • 淡出区域可能遮挡内容
  • 不适合需要回溯历史的场景

方案 D: 智能混合模式
创新方案
设计理念：普通行自动换行，超长行（如表格、长命令）独立横向滚动。
参考产品：现代代码编辑器（VS Code、WebStorm）的终端输出处理

优势

  • 兼顾可读性和格式保持
  • 普通内容自动换行，阅读舒适
  • 超长行独立滚动，不影响整体布局
  • 智能识别，用户体验最佳

劣势

  • 实现复杂度较高
  • 需要智能判断"长行"阈值
  • 视觉风格不统一
  • 可能增加渲染开销

🎯 最终建议

推荐方案 A（垂直滚动 + 自动换行）作为默认方案。

理由：
1. 符合大多数用户的阅读习惯，无需横向滚动
2. 业界主流方案（Wave Terminal、Hyper、VS Code 均采用）
3. 实现简单，性能优秀
4. 适合 TCommander 的多会话管理场景，快速浏览多个会话输出

备选方案：
- 如果用户强烈要求保持终端格式 → 方案 B
- 如果追求极致视觉效果 → 方案 C
- 如果愿意投入开发成本 → 方案 D

可优化点：
- 增加"复制原始格式"按钮，解决自动换行后的复制问题
- 提供设置选项，允许用户切换不同预览模式
- 卡片宽度建议调整到 600-640px，提供更舒适的阅读空间
